# Ydrop

**一个把语音、文字、截图和链接快速收进来，再用 AI 自动整理成行动信息的个人助手。**

**你负责随手丢，Ydrop 负责转写、归类、提取时间地点、生成提醒和日程。**

## 它在解决什么问题

很多记录工具只负责“收”，但不会真正“整理”。
灵感、待办、语音备忘、截图、网址、约会时间、地点线索，往往散落在不同地方，最后不是忘了，就是找不到。

**Ydrop 想做的是一条完整链路：**
从闪念采集，到 AI 理解，再到提醒和日程化，让碎片信息逐步变成可执行的信息。

## 现在已经能做什么

- **语音记录并自动转文字**：录完后自动走 relay + 豆包转写链路，保留原始录音
- **文字快速记录**：像原生闪念一样随手记下想法
- **自动同步到 NAS**：记录会同步到你的 WebDAV / NAS，不只留在手机里
- **支持类型和重要程度管理**：待办、任务、提醒、普通一眼区分
- **可编辑、删除、重新同步**：记录不是一次性输入，后面还能持续整理
- **支持快捷入口**：可以从快捷开关和悬浮侧边把手发起快速记录
- **支持中转外链 + 豆包转写**：语音可以进入后续 AI 整理流程

## 它和普通便签有什么不一样

- **原始记录不会丢**：AI 整理不会覆盖你最初输入的内容
- **语音不是黑盒**：录音、转写文本、同步记录都会保留
- **目标不是“存起来”而是“整理出来”**：记录最终会进入提醒、日程和行动化链路
- **数据可控**：支持 NAS / WebDAV，同步链路掌握在自己手里

## 正在推进

- **相似记录自动归并整理**  
  相同标题、相近内容或重复想法，交给 AI 自动整合，但不改动你的原始记录。

- **时间与地点线索深挖**  
  从语音、文字、截图和链接里提取时间、地点、人物等信息，自动生成日程候选。

- **提醒系统**  
  在 Ydrop 内形成提醒，也支持后续通过聊天软件推送给你。

- **截图与链接理解**  
  不只是记文字和语音，截图和网页链接也能进入整理链路。

- **更接近原生闪念的体验**  
  继续优化一键录音和一键打字逻辑，让记录动作更短、更顺、更像系统级闪念入口。

## 当前技术结构

- Android / Jetpack Compose
- Room 本地存储
- WebDAV / NAS 同步
- relay 中转服务
- 豆包语音转写

## 仓库结构

- `app/` Android 客户端
- `relay_service/` 临时音频中转服务
- `WALKTHROUGH.md` 开发走查与里程碑
